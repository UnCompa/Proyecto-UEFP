Welcome to the development team of the Unidad Educativa Fiscal Pichincha (UEFP)! This guide will provide you with an overview of the project and the resources needed to contribute to the development and maintenance of our website.

For more details and to collaborate on the project, visit our repository on [GitHub](https://github.com/UnCompa/Proyecto-UEFP).

## Project Overview

Our website is built with **Astro JS**, a web framework that allows creating fast and efficient pages to display content dynamically.

For more information about Astro JS, visit the official [Astro JS](https://astro.build/) page.

### Necessary Requirements

To contribute to the project and maintain the quality of the site, it is important to have the following knowledge and tools:

- Proficiency in HTML, CSS, and JavaScript
- Familiarity with React
- Knowledge of Astro JS
- Experience with version control using **Git**
- Competence in using the terminal

With these requirements, you will be able to actively participate in the development and continuous improvement of our project.

## How to Start

To start contributing, you will first need to clone the project repository. You can do this by running the following command in the terminal (Git is required):

```bash
git clone git@github.com:UnCompa/Proyecto-UEFP.git
```
This will clone the repository so you can start working on it.

#### Public Folder

This folder contains static resources used on the website, such as images and videos.

##### Imgs

Almacena imágenes principales utilizadas en la web.

##### Videos

Carpeta que almacena videos utilizados en la web.

##### Svg

Se guardan imágenes de tipo SVG utilizadas en la web.

#### Carpeta src

Esta carpeta contiene el código fuente de la aplicación, organizado en diferentes subdirectorios:

##### Components

Contiene componentes reutilizables utilizados en diferentes partes de la página.

##### i18n

Almacena las traducciones para español e inglés de la página.

##### Icons

Un componente de React que exporta iconos en formato SVG para su uso en la página.

##### Layouts

Aquí se encuentran las plantillas que proporcionan una base reutilizable para diferentes casos de uso.

##### Lib

Contiene bibliotecas útiles, como la biblioteca para utilizar Supabase

##### Meta

Contiene componentes con información necesaria para la indexación de la página.

##### Pages

La carpeta más importante, que contiene las rutas a las diferentes páginas de la web.

##### Schema

Carpeta para definir esquemas de datos.

##### Store

Almacena archivos relacionados con el manejo del estado de la aplicación.

##### Utils

Esta carpeta contiene utilidades varias utilizadas en el proyecto.
